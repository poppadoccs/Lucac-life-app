# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Foundation.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Context: Why Phase 1 Has Minimal UI Surface

Phase 1 is a pure refactoring and hardening phase. The eight requirements
(FOUND-01 through FOUND-06, SEC-01, SEC-02, CLEAN-02, AI-02) produce no
new screens, no new pages, and no new user-facing features. The existing
app visual appearance does not change.

The UI contract for this phase covers:
1. The stop button added to read-aloud surfaces (FOUND-06 / CLEAN-02)
2. The AI assistant visibility change — it now renders for all non-guest profiles (AI-02)
3. The permission-denied toast triggered by canWrite guard failures (SEC-01)
4. The private event indicator when an admin creates a private event (SEC-02)

All other visual surfaces (game screens, calendar, food tab, budget tab, etc.)
are untouched and carry forward from v25 unchanged.

## Color

The app has three themes (Skylight, Cozyla, FamilyWall). All new UI elements
must use `V.*` tokens — never hardcoded hex values. The 60/30/10 split maps
to the existing token structure.

| Role | V.* Token | Skylight | Cozyla | FamilyWall | Usage |
|------|-----------|----------|--------|------------|-------|
| Dominant (60%) | `V.bgApp` | #f8f9fb | #fdf6ee | #f0f4f8 | App background |
| Secondary (30%) | `V.bgCard` | #ffffff | #fffaf3 | #ffffff | Cards, panels, nav |
| Accent (10%) | `V.accent` | #f59e0b | #e87f5f | #1e3a6e | Reserved elements below |
| Destructive | `V.danger` | #dc2626 | #c4453a | #c0392b | Destructive actions only |
| Success semantic | `V.success` | #16a34a | #5a9e6f | #1a7a42 | Confirmation, positive feedback |

**Accent reserved for (explicit list):**
- Active tab indicator in bottom nav
- Primary CTA buttons ("Save", "Add", "Unlock")
- Star count display on kid profile
- Section headings that name a feature (e.g., "LUCAC Life" app title)
- Stop button background when actively speaking (reading aloud is in progress)
- Private event lock badge background (`${V.accent}18` background with `V.accent` text — matches existing badge pattern)

**Accent is NOT used for:**
- Body text
- Secondary buttons (use `V.bgElevated` + `V.textSecondary`)
- Disabled states
- Error states (use `V.danger`)

**Colorblind constraint (CLAUDE.md — Alex is deutan colorblind):**
Every state communicated by color MUST also have a label, icon, or pattern.
- Private event: lock icon + "Private" label + accent background — never color alone
- Permission denied: "Permission denied" text in toast — never red border alone
- AI assistant role indicator: role label ("Admin", "Parent", "Kid") + icon — never color alone
- Stop button active state: "Stop" label + icon — never color change alone

## Component Inventory (Phase 1 New UI Elements)

These are the only new UI elements Phase 1 introduces. All must follow existing
inline-style patterns using `V.*` tokens.

### 1. Stop Button (FOUND-06 / CLEAN-02)

Appears in: HomeworkHelper read-aloud, ReadingGame read-aloud.

```
Anatomy:
  [icon] [label]
  🔊 "Stop"  →  when speaking (accent background)
  🔈 "Read"  →  when idle (secondary button style)

Style (active/speaking):
  background: V.accent
  color: #fff
  border: none
  borderRadius: V.r2
  padding: "10px 16px"
  fontWeight: 700
  fontSize: 13
  minHeight: 44px
  cursor: pointer

Style (idle):
  background: V.bgElevated
  color: V.textSecondary
  border: `1px solid ${V.borderSubtle}`
  borderRadius: V.r2
  padding: "10px 16px"
  fontSize: 13
  minHeight: 44px
  cursor: pointer
```

### 2. Permission-Denied Toast (SEC-01)

Appears in: App.jsx fbSet wrapper — triggered when canWrite returns false.

Use existing `showToast` function. Call with:
```javascript
showToast("Permission denied", "error")
```

The existing toast renders with `V.danger` background. No new component needed.

### 3. Private Event Badge (SEC-02)

Appears in: Event creation form — admin-only toggle.

```
Label: "🔒 Private"
Style:
  fontSize: 11
  fontWeight: 700
  color: V.accent
  background: `${V.accent}18`
  padding: "2px 8px"
  borderRadius: 10
  display: inline-flex
  alignItems: center
  gap: 4
```

Toggle control: checkbox or tap-to-toggle button. Must show both lock icon AND
"Private" text (colorblind rule). When toggled on, show the badge inline.

### 4. AI Assistant Role Indicator (AI-02)

Appears in: GroqAssistant floating panel header — shows which tool tier is active.

```
Admin:   👑 "Full access"
Parent:  👨‍👩‍👧 "Family access"
Kid:     🧒 "Kid mode"

Style (matches existing status badge pattern):
  fontSize: 11
  fontWeight: 700
  color: V.textDim
  background: V.bgElevated
  border: `1px solid ${V.borderSubtle}`
  padding: "2px 8px"
  borderRadius: 10
```

This indicator is informational — it tells the user what the assistant can
do for them. It does not replace the existing assistant UI structure.

## Interaction Contracts

### Read-Aloud Stop Control (FOUND-06 / CLEAN-02)

- Stop button renders when `speakText` is called and `window.speechSynthesis` exists.
- While reading: button shows "Stop reading" + icon, accent background.
- Tapping Stop: calls the stop handle returned by `speakText()`. Button reverts to idle.
- While idle: button shows "Read aloud" + icon, secondary button style.
- If `window.speechSynthesis` is undefined: button is hidden entirely (not disabled — no ghost buttons for unsupported features).

### Permission Guard Toast (SEC-01)

- canWrite check happens inside fbSet before any Firebase write.
- On block: show "error" toast via existing showToast. No modal, no confirm dialog.
- Toast auto-dismisses (existing behavior). Duration: existing default (2-3s).
- UI element that triggered the blocked write does NOT visually change — the toast is the only feedback.

### Private Event Toggle (SEC-02)

- Admin sees the toggle in the event creation/edit form.
- Non-admin profiles do not see the toggle. The field is admin-only.
- Toggle state persists via `isPrivate` field on the event object in Firebase.
- In calendar views: private events show the "🔒 Private" badge on the event pill — admin only. Non-admin profiles do not see the event at all (filtered at read time).

### AI Assistant Role Filtering (AI-02)

- GroqAssistant renders for all non-guest profiles (remove the current admin-only gate).
- Role indicator badge appears in the assistant panel header.
- Tool access is silent — the assistant does not explain what it can't do. It simply never receives those tools from the ROLE_TOOLS filter.
- Kids receive a kid-appropriate empty state prompt instead of the standard placeholder.
