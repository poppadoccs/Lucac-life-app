🎭 PLAYWRIGHT AUTOMATED TESTING WORKFLOW

How We QA This App With Playwright MCP

v25 was the first version where Playwright MCP was used for a full end-to-end audit of the live production app. Here's the workflow that was established:

Step 1: Launch Audit

Navigate Playwright to lucac-life-app.vercel.app at mobile viewport (390×844). Enter Guest Mode (no PIN needed). Systematically click through every tab.

Step 2: Screenshot Everything

Take full-page screenshots of each tab/view. Use accessibility snapshots to read the DOM tree. Check console for errors. Click into modals, dropdowns, every interactive element.

Step 3: Document Bugs

Each bug gets: severity (CRITICAL/HIGH/MOD/LOW), description, screenshot filename, root cause analysis, and proposed fix.

Step 4: Fix All Bugs

Fix directly in source. Build. Push. Wait for Vercel deploy.

Step 5: Re-Test

Run Playwright again against the live deployment. Verify each fix with before/after evidence. For admin-only features that can't be tested without PIN, verify by checking the deployed JS bundle for expected strings.

Key Commands Used

browser_navigate → browser_snapshot → browser_click → browser_take_screenshot → browser_console_messages → browser_run_code (for bundle verification)

This workflow found 8 bugs that manual testing missed — including a CRITICAL exponential event duplication bug that would have made the calendar unusable over time.

🏗️ Architectural Change: displayEvents Layer

The biggest fix in v25 was an architectural refactor of how repeat events work:

  • Before: saveEvents() expanded repeats into individual Firebase entries (52 writes for a weekly event). This caused exponential duplication.
  • After: Store ONE event at the base date with repeat metadata (repeat, repeatCount, repeatEnd). A computed displayEvents object expands them virtually at render time and deduplicates by title|time.
  • Virtual events carry _baseDk and _baseIdx so delete/edit/style operations reference the correct Firebase source.
  • All display paths use displayEvents[dk]. All write paths use raw events.
